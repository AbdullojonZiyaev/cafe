# Видеоменю для ресторана (Техническое задание)

## 🎯 Обзор проекта

Веб-приложение для демонстрации меню ресторана в формате Instagram Stories, оптимизированное для планшетов и работающее в режиме киоска. Полная поддержка PWA с offline-доступом, Service Workers для кэширования видео, и две режимы работы (Меню и Интерактивный стол).

---

## ✅ Реализованные требования ТЗ

### 1. **Архитектура и режим работы**
- ✅ **PWA**: Полная поддержка Progressive Web App с установкой на планшеты
- ✅ **Kiosk Mode**: Встроенная поддержка полноэкранного режима через Fullscreen API (админ-панель)
- ✅ **Offline-доступ**: Service Worker с Cache-First стратегией для видео и меню

### 2. **Пользовательский интерфейс**
- ✅ **Instagram Stories**: Зацикленные видео во весь экран с автоматическим переходом (6.5 сек)
- ✅ **Верхний блок**: Название блюда, описание, иконка информации (ℹ) для подробностей
- ✅ **Центральный блок**: Интерактивные карточки товаров в горизонтальном свайпе (3 видимые)
- ✅ **Нижний блок**: Навигация (влево/вправо), корзина с индикатором, кнопка заказа

### 3. **Модальное окно с информацией**
- ✅ **Открытие**: Нажатие на иконку (ℹ) на карточке или свайп вверх (подготовлено)
- ✅ **Содержимое**: 
  - Полное описание блюда
  - Вес/выход (граммы)
  - КБЖУ (Калории, Белки, Жиры, Углеводы)
  - Список аллергенов с предупреждением ⚠️
- ✅ **Закрытие**: Крестик (✕) или клик вне окна

### 4. **Корзина и оформление заказа**
- ✅ **Видимость**: Иконка корзины в нижнем углу с счетчиком товаров и суммой
- ✅ **Управление**: Развертывающееся меню с списком товаров
- ✅ **Кнопка заказа**: Доступна только в режиме "Интерактивный стол"
- ✅ **Уведомление**: "Ваш заказ принят! Официант скоро подойдет" с автоскрытием

### 5. **Функциональные интеграции**
- ✅ **Service Workers**: Кэширование видео, постеров, и меню данных
- ✅ **PWA Manifest**: `manifest.json` с конфигурацией для установки
- ✅ **Meta-теги**: Поддержка iOS и Android (apple-web-app-capable)

### 6. **Административная панель**
- ✅ **Скрытый доступ**: Тройное нажатие на экран открывает админ-панель
- ✅ **Функции**:
  - Включение/выключение Kiosk Mode (Fullscreen)
  - Переключение между режимами: "Меню" (официант ведет) и "Интерактивный стол" (самостоятельный заказ)
  - Быстрое управление режимами без перезагрузки

### 7. **Два режима работы**

#### **Режим "Меню"** (по умолчанию)
- Корзина видна в виде "блокнота"
- Нет кнопки "Заказать"
- Гость просто листает и выбирает, официант сам пробивает заказ

#### **Режим "Интерактивный стол"**
- Полнофункциональная корзина
- Кнопка "Заказать" отправляет заказ
- Гость самостоятельно собирает и подтверждает заказ

## 🎮 Управление

| Действие | Результат |
|----------|-----------|
| Нажатие левую половину экрана | Предыдущая история |
| Нажатие правую половину экрана | Следующая история |
| Длительное нажатие (180мс) | Пауза истории (плавное возобновление) |
| Нажатие на ℹ иконку | Открыть модальное окно с деталями |
| Нажатие "В корзину" | Добавить товар в корзину |
| Нажатие на заголовок корзины | Развернуть/свернуть корзину |
| Нажатие "Заказать" | Отправить заказ (только в режиме "Интерактивный стол") |
| Тройное нажатие на экран | Открыть админ-панель |

## 📦 Service Worker и кэширование

### Что кэшируется
- Мобильные видео (story-*-mobile.mp4)
- HD видео (story-*.mp4)
- Постеры (story-*.jpg)
- Манифест (manifest.json)
- Основные ресурсы приложения

### Стратегия
- **Cache-First**: Проверяется кэш сначала, затем сеть
- **Fallback**: Если ресурс недоступен, возвращается ошибка 503
- **Очистка**: При обновлении Service Worker удаляются старые кэши

## 🌐 PWA установка

### iOS
1. Safari → Открыть "Видеоменю"
2. Поделиться → На экран Домой
3. Приложение установлено в полноэкранном режиме

### Android
1. Chrome → Меню → Установить приложение
2. Приложение добавляется на главный экран
3. Откроется в полноэкранном режиме как приложение

---

## 🔐 Kiosk Mode

### Активация (для администратора)
1. Нажмите 3 раза на экран (быстро)
2. Нажмите "Киоск-режим (Fullscreen)"
3. Приложение перейдет в полноэкранный режим

### Деактивация
1. Нажмите 3 раза на экран
2. Нажмите "Выход из Fullscreen"

## 📝 Примечания и рекомендации

1. **Видео**: Все видео должны быть в формате H.264 MP4 с базовым профилем для совместимости с мобильными браузерами
2. **Планшеты**: Тестировано на iPad (12.9") и Samsung Galaxy Tab (10")
3. **Offline**: Service Worker требует HTTPS в production (или localhost для разработки)
4. **Admin код**: Тройное нажатие открывает админ-панель - может быть заменено на конкретный код в будущем
5. **POS интеграция**: Готова к добавлению webhook'ов для отправки заказов в системы типа iiko, r-keeper
